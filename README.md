# 🎡 Колесо Фортуны — инструкция по развёртыванию

Три компонента:
- **Google Sheet** — база данных (кто крутил, какой приз, когда)
- **Google Apps Script** — бэкенд (определяет приз, пишет в таблицу, отправляет приз в чат)
- **GitHub Pages** — фронтенд (красивое колесо, которое открывается в Telegram)

Всё бесплатно. Время на настройку: ~30 минут.

---

## Шаг 1. Создайте Google Таблицу

1. Откройте https://drive.google.com и нажмите **Создать → Google Таблицы**
2. Переименуйте в «Колесо Фортуны»

Оставьте пустой — скрипт сам создаст нужный лист и заголовки при первом запуске.

---

## Шаг 2. Добавьте Apps Script

1. В вашей таблице: меню **Расширения → Apps Script**
2. Откроется редактор. Удалите весь код по умолчанию
3. Откройте файл `apps_script.gs` из этой поставки и скопируйте весь код в редактор
4. В начале файла заполните **три константы**:
   ```javascript
   const BOT_TOKEN    = '123456789:AAH...';              // токен от @BotFather
   const BOT_USERNAME = 'home_fort_bot';                 // ваше имя бота
   const WEB_APP_URL  = 'https://ВАШ_ЛОГИН.github.io/wheel/'; // пока оставьте как есть, обновим в шаге 5
   ```
5. Нажмите **💾 (Сохранить)** или `Ctrl+S`

### Проверка токена бота
В верхней панели выберите функцию `testGetMe` и нажмите **▶ Выполнить**. 
При первом запуске Google попросит разрешения — согласитесь.
В логах (View → Logs или `Ctrl+Enter`) должно появиться что-то вида:
```
{"ok":true,"result":{"id":...,"username":"home_fort_bot",...}}
```
Если `"ok":false` — токен неверный.

---

## Шаг 3. Разверните Apps Script как Web App

1. В правом верхнем углу: **Deploy → New deployment**
2. Нажмите иконку шестерёнки ⚙️ → выберите **Web app**
3. Заполните:
   - **Description:** `Wheel backend v1`
   - **Execute as:** `Me (ваш email)`
   - **Who has access:** `Anyone` ← ОБЯЗАТЕЛЬНО
4. Нажмите **Deploy**
5. Скопируйте **Web app URL** — он вида `https://script.google.com/macros/s/AK.../exec`

Сохраните этот URL — он понадобится дважды.

---

## Шаг 4. Подключите webhook Telegram

Бот должен знать, куда отправлять сообщения пользователей. В редакторе Apps Script выберите функцию `setupWebhook` и нажмите **▶ Выполнить**.

В логах должно быть:
```
{"ok":true,"result":true,"description":"Webhook was set"}
```

Если видите ошибку — проверьте токен бота и что скрипт успешно развёрнут.

---

## Шаг 5. Залейте фронтенд на GitHub Pages

### Если нет GitHub-аккаунта
Зарегистрируйтесь на https://github.com — бесплатно.

### Создайте репозиторий
1. Нажмите **+ → New repository**
2. Имя: `wheel` (или любое другое)
3. Выберите **Public**
4. Поставьте галку **Add a README file**
5. **Create repository**

### Настройте GitHub Pages
1. В репозитории: **Settings → Pages** (в левом меню)
2. Source: **Deploy from a branch**
3. Branch: **main** / **(root)**
4. **Save**
5. Обновите страницу через минуту — сверху появится URL вида `https://ваш_логин.github.io/wheel/`

### Подготовьте и загрузите index.html
1. Откройте `index.html` из поставки
2. Найдите строку:
   ```javascript
   const BACKEND_URL = 'https://script.google.com/macros/s/AKfycbz2yd22iUnmzTW7zcec4Ttsr7DRXeNeo5J0YktLANJMvfSAKGV8vm41g-7hxtW25q0O/exec';
   ```
3. Замените на Web App URL из шага 3
4. Сохраните файл
5. В репозитории на GitHub: **Add file → Upload files**
6. Перетащите `index.html`
7. **Commit changes**

Откройте в браузере `https://ваш_логин.github.io/wheel/` — должно показаться колесо с ошибкой «Это приложение нужно открывать через Telegram». Это нормально — значит фронтенд работает.

## Шаг 6. Обновите URL в Apps Script

Теперь когда известен адрес фронтенда, нужно прописать его в бэкенд.

1. Вернитесь в редактор Apps Script
2. В константах вверху обновите:
   ```javascript
   const WEB_APP_URL = 'https://ваш_логин.github.io/wheel/';
   ```
3. Сохраните (`Ctrl+S`)
4. **Deploy → Manage deployments**
5. Нажмите на значок карандаша ✏️ у активного развёртывания
6. Version: **New version** → **Deploy**

> ⚠️ Если получите **новый** URL после деплоя — заново запустите `setupWebhook` (шаг 4).

2. Напишите вашему боту `@home_fort_bot` команду `/spin`
3. Должна прилететь кнопка «🎯 Крутить колесо»

## 🔧 Управление и операции

### Выдать пользователю новый шанс крутить
Откройте Google Sheet, найдите строку с его `user_id` в листе `spins` и удалите её. При следующем `/spin` он сможет крутить снова.

### Изменить призы или вероятности
В файле `apps_script.gs` найдите массив `PRIZES` и поправьте:
- `label` — название (влияет на текст в чате и в таблице)
- `weight` — относительная вероятность (чем больше, тем чаще выпадает)
- `tier` — визуальный уровень: `common` / `rare` / `super`

Если меняете названия призов, поправьте их **и в `index.html` тоже** (массив `PRIZES`) — иначе анимация может остановиться не на том секторе. `id` должны совпадать!

После правок:
- Apps Script: Deploy → Manage deployments → Edit → New version → Deploy
- GitHub: загрузите обновлённый `index.html`

### Изменить текст сообщения при выигрыше
В `apps_script.gs` правьте константу `WIN_MESSAGE_TEMPLATE`. Плейсхолдер `{приз}` подставляется автоматически.

### Посмотреть логи / отладка
Apps Script → слева **Executions** — увидите все обращения к бэкенду, время выполнения, ошибки.

### Снять webhook (если нужно)
Запустите в Apps Script функцию `removeWebhook`.

## ❓ Что может пойти не так

**Бот не отвечает на `/spin`**
- Проверьте, выполнена ли `setupWebhook` (логи должны показать `"ok":true`)
- Запустите `getWebhookInfo` — посмотрите, какой URL сейчас установлен
- Возможно, при `Manage deployments → New version` сменился URL — повторите `setupWebhook`

**В Mini App ошибка «Это приложение нужно открывать через Telegram»**
- Значит вы открыли его по прямой ссылке в браузере. Открывайте через кнопку в чате бота — это нормально

**В Mini App «Ошибка загрузки: HTTP ...»**
- URL бэкенда в `index.html` не совпадает с Web App URL из Apps Script
- Или Web App развёрнут с правами не «Anyone», а «Only myself» — исправьте

**«Invalid initData signature»**
- Токен бота в Apps Script не совпадает с реальным. Проверьте `BOT_TOKEN` — там не должно быть пробелов и должен быть точно как дал BotFather

**Приз прилетает в чат, но в таблице пусто**
- Проверьте, что Apps Script запускается от вашей учётки (Execute as: Me) и что у него доступ к таблице
